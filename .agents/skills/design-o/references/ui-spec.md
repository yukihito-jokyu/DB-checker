# UI Spec

画面実装前に、ユーザー体験とコンポーネント境界を具体化する。必要に応じて `frontend-implementation` skill も読む。

## 出力

- 画面目的
- 主要ユーザーフロー
- コンポーネント構成
- 状態: loading, empty, error, success, disabled
- 入力/操作/キーボード操作
- データ表示ルール
- ローカル保存: 保存キー、初期値、不正値復旧、既存stateとの同期
- アクセシビリティ観点
- レスポンシブ観点
- 実装対象と対象外

## ルール

- 実装者が迷う状態分岐を残さない。
- 見た目だけでなく、データ取得、エラー、再試行、操作不能状態を含める。
- backend契約が必要なら `backend-spec.md` と接続する。
